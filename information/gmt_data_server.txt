# Master table with information about all the remote data sets available on the GMT Data server
# Changes related to data on the gmtserver must be added into this file before they are available to users.
# Set your editor to TAB width = 4 for alignments.
#
# Updated May 26, 2020
# Note: The crontab script srv_git_update.sh will count non-commented lines and write that count, then append
#       the non-commented lines of this file and place it in the data directory. It is that file that is synced by users.
#
# Here is a longer explanation of the columns below
# Dir:		The directory on gmtserver (i.e., under /home/gmtserver/data) where files are found
# Name:		File names (with extensions) or Tile directory (no extension) for each data set
# Inc:		The grid increment of the data set in xxy format (xx = numbers, y unit among d|m|s)
# Reg:		Grid/image registration: g for gridline and p for pixel registration
# Size:		Human-readable size of the data set (e.g., 34M)
# Tile:		0 for completed grids, nonzero indicates Name is a directory with tiles of this size in degrees
# Tag:		String used to make tile grid names, e.g., N34E135.ER01m.jp2
# TGrid:	NetCDF pixel-registered grid indicating where we have tiles (1 = tile, 0 = no tile).  - if all tiles are present
# Backfill:	Background blend grid to use in areas where there are no tiles, or - if no background is needed
# Remark:	Final sentence giving info on the data set and reference to data source
#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# Dir			             Name					Inc	Reg	Size  Tile Tag 		Tgrid 						Backfill    	Remark
#
# Symbolic links for convenience and backwards compatibility with GMT <= 6.0.0; these points to the gridline-registered versions below (except 15s which is pixel only)
#
